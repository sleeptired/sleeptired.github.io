문제코드
```cpp
    std::vector<PotionRecipe> Recipes_Results;
    for (int i = 0; i < this->recipes_.size(); i++)
    {
        bool isMatch = false;
        if (name == recipes_[i].GetpotionName())
        {
            isMatch = true;
        }
        else
        {
            for (int j = 0; j < this->recipes_.size(); j++)
            {
                for (int z = 0; z < this->recipes_[j].Getingredients().size(); z++)
                {
                    if (this->recipes_[j].Getingredients()[z] == name)
                    {
                        isMatch = true;
                        break;
                    }
                }
            }

        }

        if (isMatch)
        {
            Recipes_Results.push_back(recipes_[i]);
        }
    }
```
컴퓨터가 작동한 방식 (물약 "321"을 검사할 때):

컴퓨터: "자, 이제 두 번째 물약인 '321' (인덱스 i=1) 검사할게."

이름 확인: "이름이 '1'이 아니네. else로 넘어가자."

재료 확인: "어? j 루프가 또 있네? 0번 레시피('123')부터 다시 뒤져라!"

대참사 발생: 0번 레시피인 '123'의 재료 중에 '1'이 있습니다. 그래서 isMatch가 true가 됩니다.

결과: 물약 '321'은 자기 재료에 '1'이 없는데도, 물약 '123' 덕분에 묻어가기로 검색 완료 처리가 되어버린 것입니다!
