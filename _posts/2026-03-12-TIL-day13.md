> ## 💡 [Pro-Tip] C++ 메모리 누수(Memory Leak) 잡아내기
>
> 12일차에 배웠던 메모리 누수를 더 쉽게 잡아내는 코드  
> 동적 할당이 많은 게임 개발에서 메모리 누수는 치명적입니다. 아래 코드를 `main` 함수 최상단에 넣으면 프로그램 종료 시 출력 창에 누수 여부를 친절하게 알려줍니다. **선택이 아닌 필수**입니다!  
> 
> ```cpp
> #include <crtdbg.h>
> 
> int main() {
>     // 프로그램 종료 시 메모리 누수 리포트를 출력창에 표시함
>     _CrtSetDbgFlag(_CRTDBG_ALLOC_MEM_DF | _CRTDBG_LEAK_CHECK_DF);
>     // 메모리 릭 추적.
>	    // _CrtSetBreakAlloc(161);//디버그에 찍힌 블럭 번호를 입력
>		  // 메모리 릭 지점에 자동으로 중단점을 걸어주는 함수.
>     
>     /* 로직 시작 */
> }
> ```
{: .prompt-danger }

2) PotionRecipe 멤버 변수 캡슐화  
PotionRecipe 내 데이터가 외부에서 무분별하게 수정되는 것을 방지하기 위해 public 접근 제어자를 private으로 변경하고 캡슐화를 적용했습니다.  

```diff
 class PotionRecipe {
- public:
-    std::string potionName;
-    std::vector<std::string> ingredients;
+ private:
+    std::string potionName;
+    std::vector<std::string> ingredients;

+ public:
+    PotionRecipe(const std::string& name, const std::vector<std::string>& ingredients)
+        : potionName(name), ingredients(ingredients) {
+    }
 };
```
**멤버 변수를 private로 선언을 했기때문에 다른 곳에서 접근을 못하기 때문에 오류가 생겨 멤버 함수 get으로 값을 받아오게 구현**

## C++ 강의

### 1.생성자
● 클래스를 자료형으로 포인터를 선언하고 할당을 안받고 그냥 선언만 하면 생성자는 호출안된다.  
(주소값만 저장한다고 하는거지 객체가 만들어지는거는 아니다.)  

● 멤버변수로 const나 참조자가 있을 때 생성하자마자 같이 초기화를 시켜줘야하기때문에 이니셜라이즈로 무조건 초기화 시켜줘야한다.  

● 생성자가 정의되어 있는데 클래스를 배열로 받고 싶을때  
(디폴트 생성자를 만들면 되지만 못만들 경우)  

```cpp
//1번 방법
//생성자가 선언된 클래스를 배열로 선언하는 방법
int a = 10;
// 20개를 다 적기 힘들지만, 이렇게 하면 기본 생성자 없이 배열 선언이 가능합니다.
Test A[3] = { Test(a), Test(a), Test(a) };

//2번 방법
// 주소값(포인터) 20개를 담을 공간을 만듭니다. (이때는 Test의 생성자가 필요 없습니다.)
Test** arr = new Test * [20];

// 반복문을 돌며 내가 원하는 값(a)을 넣어 진짜 객체들을 하나씩 생성합니다.
for (int i = 0; i < 20; ++i)
{
		arr[i] = new Test(a); // 생성자 호출 성공!

}

//3번 방법(제일 쉽고 많이 씀)
int a = 10;
// 20개의 칸을 만들고, 모든 칸을 Test(a) 객체로 복사해서 채웁니다.
std::vector<Test> v(20, Test(a));
```
