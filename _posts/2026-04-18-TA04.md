# 프로시듀얼 셰이더 응용

## 과제
**프로시듀얼 셰이더(Procedural Shader) 개념을 응용하여 나만의 패턴 직접 구현하기**

이번 과제에서는 외부 이미지 텍스처를 사용하지 않고, 오직 수학적 계산만으로 무늬를 만들어내는 **프로시듀얼 셰이더**를 활용해 볼 예정입니다. 

특히 스타크래프트 2의 핵 미사일 이펙트나 FPS 게임의 조준선, **바닥 장판 공격 또는 범위 및 경고 표시**를 프로시듀얼 방식으로 구현해 보고자 합니다.

> **프로시듀얼 셰이더의 핵심**
> 프로시듀얼 방식이란, **수학적인 계산을 통해 화면에 0(검정)과 1(하양)로 이루어진 마스크(모양)를 깎아내는 과정**입니다.

### 1. 머티리얼 세팅

![사진](/assets/img/TA/TA04_Setting.png) 

* **Blend Mode (블렌드 모드):** `Opaque` -> **`Translucent`**
  * **변경 이유:** 배경의 땅바닥을 완전히 가리지 않고, 우리가 만든 장판 모양만 화면에 띄운 채 그 외의 배경은 비쳐 보이게 만들기 위함입니다.
* **Shading Model (셰이딩 모델):** `Default Lit` -> **`Unlit` (빛의 영향을 받지 않음)**
  * **변경 이유:** 타겟팅 장판은 맵의 조명이나 그림자에 가려져 어두워지면 안 됩니다. 어떤 환경에서든 플레이어의 눈에 잘 띄어야 하므로 Unlit으로 설정합니다.

<br>

#### Opacity (불투명도) 기능의 원리
블렌드 모드를 `Translucent`로 변경하면 머티리얼 노드에서 **Opacity** 핀이 활성화됩니다. 

이 Opacity 핀에 들어가는 데이터 값에 따라 투명도가 결정됩니다.
* **`0` (검은색):** 투명해져서 배경이 그대로 비쳐 보입니다.
* **`1` (흰색):** 불투명해져서 우리가 지정한 색상이 화면에 출력됩니다.

### 2. 중심 이동과 거리 구하기 (원형 그라데이션 만들기)

프로시듀얼로 원을 그리기 위해서는 **화면 정중앙에서부터 거리가 얼마나 떨어져 있는가**를 구하는 것이 핵심입니다.

#### 중심점 이동하기 (Subtract 노드)
기본적으로 화면의 UV 좌표는 **왼쪽 위 모서리가 `(0, 0)`**으로 시작합니다. 하지만 우리가 만들 장판(원)은 **화면 정중앙**에 있어야 합니다.
* **해결 방법:** `TexCoord(UV)`에서 `0.5`를 빼줍니다. (Subtract 노드 사용)
* **결과:** 좌표계가 이동하여, 이제 화면의 **정중앙이 `(0, 0)`**이 됩니다. (범위: `-0.5` ~ `+0.5`)

#### 중앙으로부터의 거리 구하기 (VectorLength 노드)
정중앙이 `(0,0)`이 되었다면, 이제 화면상의 모든 픽셀들이 **중앙으로부터 얼만큼 떨어져 있는지(거리)**를 하나의 숫자로 알아내야 합니다. 이때 사용하는 것이 바로 **`VectorLength`** 노드입니다.

* **VectorLength 노드의 역할:** 픽셀까지의 '직선거리'를 계산해 줍니다.
  * 중앙점 `(0,0)`의 거리는 **0 (검은색)**
  * 모서리로 갈수록 거리가 멀어짐 **~1 (점점 하얀색으로 변함)**
* **결과:** 이 결과를 머티리얼의 색상(Emissive Color)에 연결해 보면, 중심은 새카맣고 밖으로 갈수록 하얗게 퍼지는 **방사형 그라데이션**이 나타납니다.

> **참고: VectorLength (V2 vs V3)의 차이**
> * **V2 (Vector 2):** 2D 평면(X, Y 혹은 U, V)에서의 거리를 구합니다. 지금처럼 화면상에 2D 장판(원)을 그릴 때 사용합니다.
> * **V3 (Vector 3):** 3D 공간(X, Y, Z)에서의 거리를 구합니다. 3D 입체적인 구(Sphere) 형태의 이펙트나 구형 마스크를 만들 때 사용합니다.

### 3. 테두리 구현

중심점으로부터의 거리(`VectorLength`)를 구했다면, 이제 이 값을 이용해 명확한 경계선이 있는 **원**을 만들고, 그 속을 파내어 범위 공격 표시의 **테두리**를 구현해보겠습니다.

<br>

#### `If` 노드를 이용해 "큰 원" 만들기

![사진](/assets/img/TA/TA04_IfNode.png) 

경계선이 뚜렷한 큰 원을 만들어야 합니다. 이때 **`If` 노드**를 사용하여 특정 거리(반지름)를 기준으로 안쪽과 바깥쪽을 나눕니다.

* **`If` 노드의 역할 및 핀(Pin) 설명:**
  * **`A` (비교 대상):** `VectorLength`의 결과값 (중앙으로부터 각 픽셀까지의 거리)
  * **`B` (기준치/반지름):** 우리가 지정한 장판의 전체 크기 (예: `0.5`)
  * **`A < B` (원 안쪽):** 거리가 `0.5`보다 작다면 **`1 (흰색/불투명)`**을 출력합니다.
  * **`A >= B` (원 바깥쪽):** 거리가 `0.5`보다 크거나 같다면 **`0 (검은색/투명)`**을 출력합니다.
* **요약:** "중심에서 거리가 0.5 미만인 곳만 하얗게 칠해라"라는 논리입니다. 이를 통해 바깥의 부드러운 그라데이션이 날아가고 **완벽하게 꽉 찬 원**이 만들어집니다.

![사진](/assets/img/TA/TA04_OutCircle.png) 

<br>

#### `Subtract` 노드를 이용해 속 파내기 (큰 원 - 작은 원)
큰 원을 만들었다면, 이번엔 테두리만 남기기 위해 **작은 원을 구현해 파내는 작업**이 필요합니다.

1. **두 개의 원 준비하기:**
   * 위에서 만든 `If` 노드 세트를 사용하여, **`B` 값을 `0.45`**로 설정한 "조금 더 작은 원"을 하나 더 만듭니다. (이 `B` 값이 0.5에 가까워질수록 테두리 선이 얇아집니다.)

![사진](/assets/img/TA/TA04_InCircle.png) 

2. **`Subtract` (빼기) 연산하기:**
   * `A` 핀: 큰 원 (반지름 `0.5`)
   * `B` 핀: 작은 원 (반지름 `0.45`)
* **원리 요약 및 결과:** 반지름이 `0.5`인 큰 원의 영역(`1`)에서, 반지름이 `0.45`인 작은 원의 영역(`1`)을 뺍니다.
  * 겹치는 안쪽 중심부: `1 - 1 = 0` 
  * 안 겹치는 테두리 영역(0.45~0.5 구간): `1 - 0 = 1` **(테두리만 남음)**
  * 애초에 원 밖의 영역: `0 - 0 = 0` (그대로)

### 🎨 4단계: 색상 입히고 투명도 설정하기
이제 하얗게 뚫린 링에 색을 입히고, 나머지 검은 부분을 완전히 투명하게 날려버립니다.

색상 지정:

Constant 3Vector 노드를 생성합니다. (단축키: 3 + 클릭)

더블클릭하여 원하는 색상(예: 강렬한 빨간색)을 선택합니다.

색상과 마스크 합치기:

Multiply 노드를 생성합니다. (단축키: M + 클릭)

A 핀: 방금 만든 색상 노드 연결

B 핀: 3단계에서 완성한 링 마스크(Subtract 결과) 연결

최종 연결:

Multiply의 결과를 메인 노드의 **Emissive Color**에 연결합니다.

3단계에서 완성한 링 마스크(Subtract 결과)를 메인 노드의 **Opacity**에 바로 연결합니다.

## Step 1: 다중 동심원 (Multi-Ring) 마스크 (SC2 핵 조준점)
기존에 만드신 If 노드 두 개를 뺀 그릇(Ring Mask)을 여러 개 만듭니다.

파라미터화: 기존에 0.5와 0.45를 Constant 노드로 꽂으셨던 것을, 우클릭하여 **'Convert to Parameter'**로 바꿉니다. 이름을 OuterRadius_01과 InnerRadius_01로 지어줍니다.

두 번째 링 만들기: 이 링 마스크 세트(If 2개 + Subtract 1개)를 그대로 복사해서 빈 공간에 만듭니다.

두 번째 링 파라미터: 복사된 링의 B핀에 꽂힐 파라미터 이름을 OuterRadius_02와 InnerRadius_02로 짓고, 인스턴스에서 서로 다른 값을 주어 더 큰 원을 만듭니다. (예: 링1은 0.35/0.3, 링2는 0.5/0.45)

링 합치기: Add 노드를 꺼내서 두 링 마스크의 Subtract 결과를 합쳐줍니다. 이제 하얀색 동심원 두 개가 한 마스크에 담겼습니다.

## Step 2: 십자선 (Crosshairs) 마스크
원 마스크 외에, UV 좌표를 직접 제어해 직선형 마스크를 만듭니다.

중심 좌표 가져오기 (복습): TexCoord -> Subtract (0.5)를 하나 더 만듭니다. (또는 기존 것을 끌어옵니다.)

수직선 뽑기:

Subtract 결과에 Component Mask 노드를 연결합니다. 디테일 패널에서 R만 체크합니다. (가로축 정보만 가져옵니다.)

마스크 결과에 Abs (절댓값) 노드를 연결합니다. (중심 0을 기준으로 양쪽 두께를 똑같이 만들기 위함)

Abs 결과를 If 노드의 A에 꽂습니다.

If 노드의 B에는 0.01(Constant)을 넣습니다. (선 두께)

A < B 에 1, 나머지에 0을 넣습니다. 👉 [세로선 완성]

수평선 뽑기: 2번의 Component Mask ~ If 세트를 복사합니다. 이번엔 Component Mask 디테일에서 G만 체크합니다. 👉 [가로선 완성]

십자선 결합: Max 노드를 꺼내서 세로선 결과(A)와 가로선 결과(B)를 연결합니다. 이제 하얀색 십자선 마스크가 완성되었습니다.

## Step 3: 맥동 (Pulse) 애니메이션 (SC2 핵 조준점 핵심)
핵미사일 경고 장판처럼, 전체 모양의 크기가 두근두근 조절되게 만듭니다. 이 애니메이션은 100% 수학으로 처리합니다.

사인파 생성: 빈 공간에 Time 노드와 Sine 노드를 꺼내 연결합니다.

범위 조절 (Remap): Sine은 -1 ~ 1 사이를 반복합니다. 크기가 너무 크거나 마스크가 사라지게 만듭니다. 이를 1.0 ~ 1.1 정도로 살짝 팽창하게 리맵합니다.

RemapValueRange 노드를 생성합니다.

Input Low: -1.0, Input High: 1.0

Output Low: 1.0 (기본 크기), Output High: 1.1 (최대 팽창 크기)

마스크에 적용 (핵심): 기존에 회원님이 만드신 VectorLength 노드의 결과 핀을 마우스 우클릭으로 떼어냅니다.

새로 만든 Multiply 노드를 꺼냅니다.

A 핀: VectorLength 결과

B 핀: 아까 만든 RemapValueRange 결과

이 Multiply의 최종 결과를 기존 위아래 두 If 노드의 A핀에 꽂아줍니다.

이제 모든 모양(링, 나중에 합칠 십자선 포함)이 시간에 따라 수축과 팽창을 반복합니다.

## Step 4: 모든 마스크 결합, 색상 및 Glow 입히기
모양 다 합치기: Add 노드를 하나 꺼냅니다.

A 핀: Step 1에서 완성한 동심원 마스크 (Add 결과)

B 핀: Step 2에서 완성한 십자선 마스크 (Max 결과)

이제 복잡한 핵미사일 조준 레티클의 최종 하얀색 마스크가 완성되었습니다.

색상 및 발광 적용:

이 합쳐진 모양에 원하는 **색상(Constant 3Vector)**을 Multiply 합니다.

그 결과에 한 번 더 Multiply 해서 50.0 정도를 곱해 강력한 **발광(Glow)**을 줍니다.

최종 결과를 메인 노드의 **Emissive Color**에 꽂습니다.

투명도 적용: 1번의 최종 마스크 결합 결과(Add 노드)를 메인 노드의 Opacity 핀에 연결합니다.

## 다중원 개별 색상 지정 로직
개별 마스크 준비: 첫 번째 링(Subtract_01)과 두 번째 링(Subtract_02)이 각각 독립된 결과물로 나와 있어야 합니다.

링별 Multiply 연결: * Subtract_01 결과 ➔ Multiply (A) 연결. Multiply (B) 에는 첫 번째 색상(Vector Parameter: Color_01)을 연결합니다.

Subtract_02 결과 ➔ 또 다른 Multiply (A) 연결. Multiply (B) 에는 두 번째 색상(Vector Parameter: Color_02)을 연결합니다.

색이 입혀진 결과 합치기: * 두 개의 Multiply 결과물을 Add 노드로 합쳐줍니다.

이렇게 하면 빨간 링과 파란 링이 각각 자기 색을 가진 채로 하나의 레이어처럼 합쳐집니다.

최종 출력: 합쳐진 Add 결과를 Emissive Color에 꽂고, 투명도(Opacity)를 위해서는 모든 하얀색 마스크들을 합친 결과를 연결합니다.

## 십자선 개별 색상 지정 및 최종 결합 로직
1. 십자선 색상 입히기

마스크 찾기: 이전에 만드셨던 십자선의 최종 하얀색 마스크인 Max 노드를 찾습니다.

색상 파라미터 생성: 빈 공간을 우클릭하여 **Vector Parameter**를 생성하고, 이름을 Color_Crosshair로 지어줍니다. (원하는 십자선 색상 지정)

Multiply 연결: 새로운 Multiply 노드를 꺼내서, A핀에는 십자선의 Max 결과를 꽂고, B핀에는 **Color_Crosshair**를 꽂아줍니다.

👉 결과: 이제 십자선이 고유의 색상을 가지게 되었습니다.

2. 발광(Emissive Color)을 위해 '색칠된 결과' 합치기

Add 노드를 새로 하나 꺼냅니다.

A핀: 앞서 만들어둔 색칠된 다중원들의 합(Add 결과)을 꽂습니다.

B핀: 방금 만든 색칠된 십자선(Multiply 결과)을 꽂습니다.

이 최종 Add 결과에 Glow 강도(예: 50.0)를 곱해서 메인 노드의 Emissive Color 핀에 연결합니다.

👉 결과: 캔버스 위에 빨간색 원, 파란색 원, 노란색 십자선이 한꺼번에 빛나게 됩니다.

3. 투명도(Opacity)를 위해 '원본 하얀색 마스크' 합치기 (매우 중요 ⭐)
투명도를 결정하는 Opacity 핀에는 색상(RGB) 데이터가 들어가면 투명도가 비정상적으로 작동할 수 있습니다. 반드시 색을 칠하기 전의 순수한 0과 1(흑백) 데이터만 합쳐서 넣어야 합니다.

또 다른 Add 노드를 새로 꺼냅니다.

A핀: 색을 칠하기 전의 원본 하얀색 다중원 마스크(Add 또는 Max 결과)를 꽂습니다.

B핀: 색을 칠하기 전의 원본 하얀색 십자선 마스크(Max 결과)를 꽂습니다.

이 결과를 메인 노드의 Opacity 핀에 연결합니다.
