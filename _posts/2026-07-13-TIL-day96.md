#  TIL —  EOS 클라이언트 로그인 방식 확정기

## 한 줄 요약
DeviceID 익명 로그인을 시도하다 언리얼 엔진 스톡 플러그인의 한계를 확인하고, EAS(에픽 계정 서비스) 애플리케이션이 도메인 인증 없이도 초안(Draft) 모드로 동작함을 발견하여 `accountportal`(브라우저 로그인) 방식으로 최종 확정함. 스코프(Scope) 불일치 문제까지 잡아내어 로그인 플로우 완성.

## 오늘의 목표
* 팀원에게 `.exe`만 배포하면 별도 도구(DevAuthTool) 없이 알아서 로그인되고 방 생성/참가가 되는 구조 만들기.
* **로그인 방식 후보:** DeviceID(익명, 창 없음) vs accountportal(에픽 계정, 브라우저 창)

## 진행 내용 (시간순)

### 1. INI 자리표시자 사고 → EOS 초기화 실패 디버깅
* **현상:** 첫 실행에서 `ClientCredentials.ClientId must be an ANSI string between 1 and 64 in length` + `EOS_NotConfigured` 에러 발생.
* **원인:** `DefaultEngine.ini`의 `Artifacts`에 실제 자격증명 대신 한글 자리표시자("클라용_ClientId")가 그대로 들어있었음.
* **분석:** EOS 초기화가 실패하면 언리얼 엔진이 조용히 `Null(LAN)` 서브시스템으로 폴백(Fallback)한다는 것을 로그로 확인. (`FindSessions 시작 (LAN=1)`이 그 증거)
* **교훈:** `LAN=0/1` 로그 한 줄이 현재 어떤 서브시스템으로 돌고 있는지 알려주는 가장 빠르고 확실한 지표다.

### 2. DeviceID 로그인 시도와 좌초
목표는 "창 없는 완전 자동 로그인"이었으나 플러그인 한계로 방향을 전환함.

* **시도 내역:**
  * **Dev Portal:** GameClient 정책을 Custom으로 전환, Sessions 권한 수동 구성.
  * **INI:** `bUseEAS=false`, `bUseEOSConnect=true`
  * **코드:** `FOnlineAccountCredentials`에 `deviceid`, `externalauth` 등 여러 타입 문자열 시도 → 전부 `External Auth Token Type not specified` 실패.
* **엔진 소스 직접 분석 (`UserManagerEOS.cpp`) 결과:**
  1. `Login()`은 조건에 따라 `CallEOSConnectLogin()` 경로로 정상 분기하고 있었음.
  2. 하지만 `CallEOSConnectLogin()`은 Type 문자열을 콜론으로 파싱하는 `externalauth:<외부타입>` 형식을 강제 요구함.
  3. **결정적 원인:** 파일 전체에 `deviceid` 관련 코드가 0건. 즉, 스톡(기본) 플러그인에는 DeviceID를 생성해주는 진입점 자체가 구현되어 있지 않음.
* **우회 시도 및 포기:** SDK 직접 호출(`EOS_Connect_CreateDeviceId`)을 위해 `IOnlineSubsystemEOS::GetEOSPlatformHandle()`을 쓰려 했으나, 플러그인 구현부가 Private으로 숨겨져 있어 작업량 대비 리스크가 크다고 판단.
* **교훈 & 보충:** 스톡 OSS EOS 플러그인의 Connect(NoEAS) 경로는 Steam 등 '타 플랫폼의 네이티브 토큰' 연동을 전제로 설계되어 있습니다. 순수 DeviceID 로그인을 원한다면 커스텀 C++ 래퍼를 작성하거나, Redpoint Games의 EOS 플러그인(무료) 같은 서드파티 확장을 쓰는 것이 일반적입니다.

### 3. 방향 전환: EAS 애플리케이션 생성 성공 (도메인 없이!)
* **현상:** 이전에는 도메인 인증(DNS TXT 레코드)에서 막혔다고 판단했음.
* **해결:** 브랜드 설정을 '미완성'으로 두고도 **애플리케이션 생성 + 권한 + 연동 클라이언트 구성**이 가능함을 확인.
* **최종 상태:** 권한 ✅ (Basic Profile만) / 연동된 클라이언트 ✅ (GameClient만) / 브랜드 설정 미구성(초안)
* **결과:** '초안(Draft) 모드'에서는 도메인 인증이나 브랜드 리뷰 없이도 로그인 동작이 가능합니다. 대가로 유저에게 "확인되지 않은 앱" 노란색 경고 페이지가 하나 뜨지만(계속 진행 가능), 개발 단계에서는 전혀 문제가 되지 않습니다. (출시 전에만 리뷰를 받으면 됨)

### 4. 스코프 불일치 — `friends_list` 403 에러
* **현상:** `accountportal` 로그인 창까지 도달했으나 *"클라이언트에 다음의 범주를 요청할 수 있는 권한이 없습니다: friends_list"* 에러 발생.
* **원인:** 에디터 GUI(Project Settings)로 설정을 저장하면서 `AuthScopeFlags`에 `BasicProfile`, `FriendsList`, `Presence` 3개가 자동 기록됨. EAS 앱 설정에서는 `BasicProfile`만 허용했으므로 **요청 > 허용** 상태가 되어 튕김.
* **해결:** INI 파일에서 배열을 비우고 하나만 명시적으로 등록.

  ```ini
  !AuthScopeFlags=ClearArray
  +AuthScopeFlags=BasicProfile
  ```

> **교훈:** 게임이 요청하는 스코프와 EAS 앱이 허용하는 스코프는 100% 일치해야 합니다. (로그인 후 SDK가 자동으로 친구 목록을 조회하다 발생하는 `insufficient_scopes` 403 Warning은 로그인은 성공한 상태이므로 무시해도 무방합니다.)

### 5. 로그인 성공 & 동작 플로우 정리

| 프로세스 | 인증 방식 | 창 표시 | 비고 |
| :--- | :--- | :--- | :--- |
| **데디서버** | TrustedServer 자격증명 | 없음 | 서버는 로그인 개념 자체가 없음. (User required 해제) |
| **클라 최초 실행** | `accountportal` (브라우저) | 1회 뜸 | 에픽 계정 로그인 |
| **클라 재실행** | `persistentauth` (저장 토큰) | 없음 | 테스트 확실성을 위해 이후 정책 변경으로 제거함 |

### 6. 유령 세션 문제 (Ghost Sessions)

* **현상:** 방 목록에 'CrazyRoo Room 4/4 0ms'가 2개씩 남아있는 현상 발생.
* **원인:** 프로세스를 강제 종료하면 `DestroySession`이 실행되지 않아 EOS 백엔드에 세션/참가 슬롯이 그대로 남음.
* **해결:**
  * 잔여 UnrealEditor 프로세스 확인 및 종료 후 대기.
  * `Deinitialize()` 등 정상 종료 경로에 `DestroySession(NAME_GameSession)` 추가.

> **보충:** EOS 백엔드는 하트비트(Heartbeat) 통신이 끊기면 자동으로 세션을 청소하지만, 타임아웃까지 보통 수 분(2~3분)이 걸립니다. 개발 중 크래시로 인한 유령 세션은 이 타임아웃을 기다리는 수밖에 없습니다.

### 7. 로그인 UX 변경: 항상 물어보기

* **현상:** `persistentauth` 자동 로그인을 제거하고 매 실행마다 `accountportal`을 띄우도록 흐름 변경.
* **비고:** 단, 브라우저에 에픽 웹 로그인 쿠키가 살아있으면 비밀번호 입력 없이 '동의' 화면만 지나가게 됨. 이는 게임이 제어할 수 없는 웹 브라우저 단의 캐시 영역임.

## 오늘 배운 것 핵심

1. **OSS 추상화의 경계:** `IOnlineIdentity::Login()`의 자격증명 문자열 해석은 플러그인마다 규칙이 다르다. 막히면 추측하지 말고 엔진 소스에서 에러 문자열을 직접 검색하라.
2. **EAS 초안(Draft) 모드의 유용성:** 도메인 인증이나 브랜드 리뷰가 없어도 권한과 연동 클라이언트만 구성하면 개발/테스트용 로그인이 완벽하게 동작한다.
3. **명시적인 세션 정리:** EOS 세션은 프로세스 강제 종료 시 백엔드에 유령으로 남는다. 정상 종료 생명주기에 반드시 `DestroySession`을 심어두자.
